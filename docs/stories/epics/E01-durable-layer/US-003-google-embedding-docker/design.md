# Design

## Domain Model

We introduce a real embedding domain provider:
1. `GoogleGenAI` from `@google/genai` is initialized using an API key (`GEMINI_API_KEY`).
2. An embedding operation converts arbitrary text chunks generated by `ASTProcessor` into `768`-dimensional floating-point vectors via the `text-embedding-004` model.
3. The embedding operation handles potential API failures (e.g. rate limits or credential issues) gracefully by printing diagnostic logs, and falls back to a clean mock (768-size array of `0.01` values) if the API key is not supplied, allowing standard offline operation.

## Application Flow

- **index_code_file (command)**:
  1. AST chunking extracts semantic nodes (files, classes, functions).
  2. For each chunk, the server calls `ai.models.embedContent` to obtain a real vector.
  3. The vector is saved to `code_nodes.embedding` (768-dimensional) in PostgreSQL.
- **semantic_code_search (query)**:
  1. The user query is embedded using the same Gemini model.
  2. A cosine-distance check (`embedding <=> $1`) queries the HNSW index in PostgreSQL.
  3. Contextually relevant matches are returned.

## Interface Contract

- The MCP tools keep their existing JSON inputs/outputs unchanged.
- The environment configuration introduces:
  - `GEMINI_API_KEY` (required for real embeddings).
  - `DATABASE_URL` (standard Postgres connection string).

## Data Model

```sql
ALTER TABLE code_nodes ALTER COLUMN embedding TYPE VECTOR(768);
```
In `schema.sql`, the schema will be initialized directly with `VECTOR(768)` to make all clean installations correct out-of-the-box. An HNSW cosine operator index is defined on this column.

## UI / Platform Impact

- **Docker Compose**: Orchestrates a `pgvector/pgvector:pg16` database service (`db`) and the app service (`mcp-server`).
- **Multi-stage Dockerfile**: Prevents native C++ dependency resolution issues on deployment by compiling `tree-sitter` in a full-build stage.

## Observability

- Graceful logging on database connection attempts.
- Explicit warnings if `GEMINI_API_KEY` is not detected, detailing that the server is operating in fallback/mock mode.

## Alternatives Considered

1. **Keep VECTOR(1536) and pad or double-embed**: Upscaling a 768-vector to 1536 by repeating values or adding zeros. This wastefully doubles storage requirements and compromises index performance. Discarded in favor of changing the vector dimensions to 768.
2. **Use legacy `@google/generative-ai` package**: The older package works, but the unified `@google/genai` is the recommended SDK going forward and has a cleaner syntax.
